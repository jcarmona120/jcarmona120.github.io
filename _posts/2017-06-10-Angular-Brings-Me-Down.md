### The Angular Workshop

I was pretty excited and scared about this Angular2 + Typescript workshop I was attending this morning. The first hour of the workshop went over pretty well. The tasks were simple - install Angular, set up the AppComponent, have the website show a list of products. Boom! I had gone over some parts of the Angular Tour of the Heroes Tutorial a few days ago so I understood how to show something super basic. I was so confident that I sent my fiancee a message of victory. Turns out - that was it. And the workshop was from 9am-4pm. So 10:15 was the last time I celebrated.

This wasn't a very in-depth workshop as far as teaching goes but I won't complain because it made me dive more in-depth into documentation. And Angular has a ton of documentation. I go back and forth between anger about how poor documentation is in general and how I might be an idiot. A constant struggle. At the moment, I'm comfortably between both. I got home and decided to dive deeper. Thank you [FrontEndMasters!](www.frontendmasters.com)

Something like Angular deserves some respect. I wouldn't suggest just diving in. Part of the Angular "Guide" section is called _Architecture_. I wouldn't just trying making a house with plumbing, electricity, etc. without maybe reading a little bit about it. At least, you want to know in what order those things go in. Do I put all the walls up first? What's the best way to organize the wiring? Well, same goes for Angular. When do I use my Decorator function and how should I prepare my Services?

Well, one more thing to add on to things I have to learn. The never ending process.

### What I Learned Today

**Angular Stuff**

Some quality things from the [Building Web Apps with Angular2 course](https://frontendmasters.com/courses/angular-2/).

@Component({}) is the “Decorator” function. Decorators are basically metadata or "mixins" for the Component it's attached to. I do enjoy this feature of Angular.

The acronym - CIDER.

+ Create class
+ Import Dependencies
+ Decorate Class
+ Enhance with Composition
+ Repeat for sub-components
